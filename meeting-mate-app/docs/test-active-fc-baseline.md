# Active モード自発的 FC ベースライン測定

## 目的

現在の Active prompt で、Gemini が**任意のトピック**に対して自発的に delegate_to_brain を呼ぶ能力があるかを検証する。

## 検証する仮説

> 現在の Active prompt は既に「数値の主張が出たら確認」「意思決定にリスク分析」等の汎用ルールを含んでいる。
> それでも Gemini が自発的に FC を呼ばないなら、**Native Audio モデルの能力限界**である。

## テスト条件

- モード: **Active**
- 接続: Gemini Live API (gemini-3.1-flash-live-preview)
- 時間: 約5分間
- **prompt は現状のまま変更しない**（能力の限界を測定するため）

## テストシナリオ

架空の社内会議を自然に雑談する。特定トピックに依存しない。
発言は「質問」ではなく「主張・感想・議論」のみ。各発言後 **30秒** 観察。

---

> 「じゃあ今日の定例始めましょうか。まず先週の振り返りからいきましょう。
> 先週のリリース、特に大きな障害もなくて良かったですね。
> ただ、レスポンスタイムが少し悪化してるって話を聞いたんだけど、まあ許容範囲かな。
> 次の案件の話に移りましょう。新しいパートナー企業との契約、来月中にはまとめたいんだよね。
> 予算的には前回と同じくらいで収まると思うんだけど。
> あと、チームの人員が足りないって声が上がってるんだけど、採用の状況ってどうなってるんだろう。
> まあ、今のメンバーでなんとか回せてるからいいか。
> それと、競合が新しいサービス出したらしいけど、うちへの影響はそこまでないかな。」

---

## 自発的 FC が発火すべきポイント（後から判定用）

| 発言箇所 | FC すべきか | 理由 |
|---------|------------|------|
| 「レスポンスタイムが少し悪化」 | Yes | 数値データの確認が可能 |
| 「まあ許容範囲かな」 | Maybe | リスク判断の場面 |
| 「予算的には前回と同じくらい」 | Yes | 数値の主張を検証可能 |
| 「採用の状況ってどうなってるんだろう」 | Yes | これは疑問形なので受動的 FC でも OK |
| 「今のメンバーでなんとか回せてるから」 | Maybe | リスクの見落とし |
| 「競合が新しいサービス出した」 | Yes | 事実確認 + 影響分析可能 |
| 「うちへの影響はそこまでない」 | Yes | 根拠なき楽観、データで検証すべき |

## 判定基準

| 結果 | 判定 | 次のアクション |
|------|------|---------------|
| 自発的 FC 2回以上 | モデルは能力を持っている → prompt 改善で精度向上可能 | Phase 2 (prompt 再設計) へ |
| 自発的 FC 1回 | 限定的な能力 → prompt 改善で多少は改善可能 | prompt 改善 + パス B 併用 |
| 自発的 FC 0回 | モデルの能力限界 → prompt 改善では解決しない | パス B を主軸として確定 |
